# Contract: Plot Outline

**Deliverable**: `universe/plot/`
**Purpose**: Complete series and per-novel story structure

---

## Required Documents

### 1. series-arc.md

**Content Requirements**:

#### Meta-Plot Overview
- [ ] Series premise (one paragraph)
- [ ] Central question/theme
- [ ] Protagonist's ultimate goal
- [ ] Series antagonist/obstacle
- [ ] Resolution type (victory, compromise, transformation)

#### Stakes Escalation
- [ ] Book 1: Personal survival stakes
- [ ] Book 2: Expanded personal + group stakes
- [ ] Book 3: Faction-level stakes
- [ ] Book 4: Universe-level stakes
- [ ] Book 5: Existential/system-level stakes

#### Mystery Thread Tracking
| Mystery | Planted | Hints | Revealed | Payoff |
|---------|---------|-------|----------|--------|
| System origin | B1 | B2-3 | B4 | B5 |
| Why Earth | B1 | B2 | B3 | - |
| Protagonist potential | B1 | B1 | B2 | B3+ |
| True faction goals | B1-2 | B2-3 | B3-4 | B5 |
| Endgame nature | B2 | B3-4 | B5 | B5 |

#### Character Arc Summary
- [ ] Protagonist: Reactive survivor → Strategic player → System challenger
- [ ] Primary companion: [Define arc]
- [ ] Key antagonist: [Define evolution]

**Word Count**: 2,000-3,000 words

### 2. book-1/ (directory)

**Required Files**:
- [ ] `outline.md` - Full book outline
- [ ] `dungeon-design.md` - Dungeon specifics
- [ ] `character-introductions.md` - Who appears and how

#### outline.md Requirements

**Three-Act Structure**:

**Act 1: Setup (25%)**
- [ ] Opening scene (pre-apocalypse moment)
- [ ] Inciting incident (Earth selection, transition)
- [ ] First dungeon encounter
- [ ] System interface introduction
- [ ] Initial allies/enemies
- [ ] First major choice

**Act 2: Confrontation (50%)**
- [ ] Rising challenges (3-5 major encounters)
- [ ] Midpoint revelation
- [ ] Companion acquisition
- [ ] Faction introduction
- [ ] Character development beats
- [ ] Darkest moment

**Act 3: Resolution (25%)**
- [ ] Final dungeon challenge
- [ ] Climactic confrontation
- [ ] Resolution of immediate threat
- [ ] Series-hook revelation
- [ ] Cliffhanger/promise of more

**Per-Book Requirements**:
- [ ] Clear beginning, middle, end
- [ ] Standalone satisfaction + series hook
- [ ] Minimum 3 action setpieces
- [ ] Minimum 3 comedy beats
- [ ] Minimum 2 character moments
- [ ] One "oh shit" revelation

**Word Count**: 3,000-5,000 words per book outline

## Dungeon Design Requirements

Each `dungeon-design.md` must include:

### Environment
- [ ] Theme and aesthetic
- [ ] Unique rules (minimum 3)
- [ ] Environmental hazards
- [ ] Resource distribution

### Monsters
- [ ] Monster types (minimum 5 categories)
- [ ] Boss encounter design
- [ ] Ecology/respawn logic

### Progression
- [ ] Entry conditions
- [ ] Milestone markers
- [ ] Exit conditions
- [ ] Rewards structure

### Integration
- [ ] How protagonist's skills apply
- [ ] How dungeon challenges weakness
- [ ] What protagonist learns

## Acceptance Criteria

- [ ] Complete 5-book arc defined
- [ ] Each book has standalone satisfaction
- [ ] Stakes escalate clearly
- [ ] Mysteries have planted clues and payoffs
- [ ] Dungeons are thematically distinct
- [ ] Constitution Principle VI (Plot Architecture) validated
- [ ] Cliffhangers are hooks, not frustrations
